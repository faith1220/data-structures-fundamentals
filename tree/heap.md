# 堆与优先队列 (Heap & Priority Queue)

在前面的章节中，我们学习了“队列（Queue）”。普通的队列讲究的是**“先来后到（FIFO, First-In-First-Out）”**，就像在超市付款排队，谁先排，谁就先结账。

但是在现实生活中，还有一种排队叫**“VIP 优先”**或者**“急诊优先”**。
想象一下急诊室的候诊大厅：虽然大家都在排队，但如果突然送进来一个心脏病突发的危重病人，即使他是最后一个到的，护士也会立刻把他安排到队伍的**最前面**优先抢救。
这种**“带有优先级，谁优先级高谁先出队”**的数据结构，在计算机里就叫做——**优先队列（Priority Queue）**。

而实现优先队列最经典、最高效的底层数据结构，叫做**堆（Heap）**。

## 什么是“堆”（Heap）？

不要一听到“堆”就觉得是一堆乱糟糟的数据。在计算机科学中，堆是一种**非常特殊的二叉树**，它必须同时满足两个极其严格的条件：

### 条件一：它必须是一棵“完全二叉树”
什么叫完全二叉树？你可以理解为**“按着顺序、从上往下、从左往右，不留任何空隙”**长出来的树。

```text
      (完全二叉树)                  (非完全二叉树 - 违规了)
           1                              1
         /   \                          /   \
        2     3                        2     3
       / \                            / \      \
      4   5                          4   5      6
    (绝不留空隙，紧凑生长)          (右边留了空隙，直接长到最右边去了)
```

### 条件二：老总绝对比下属强（大顶堆 / 小顶堆）
根据你要选出“最大值优先”还是“最小值优先”，堆被分为两种：
1. **大顶堆（Max-Heap）**：任何一个父节点的值，都**大于或等于**它左右孩子的值。这棵树最大的那个数字，肯定高高在上地坐在**根节点**的位置。
2. **小顶堆（Min-Heap）**：任何一个父节点的值，都**小于或等于**它左右孩子的值。树的最顶端，坐着全局最小的数字。

> **💡 注意：** 堆**不要求**左右孩子之间谁大谁小（这跟上一章的二叉搜索树 BST 完全不同），它只要求“上下级”关系绝对分明。

## 堆的魔术：树变成了一维数组！

堆最令人拍案叫绝的设计在于：**它虽然在我们的脑子里是一棵二叉树，但在计算机的内存里，它是用一个“极其普通的数组”来存储的！**

因为堆是一棵“不仅凑、不留空隙”的完全二叉树，所以它所有的节点都可以按**“层序遍历”**的顺序，严丝合缝地塞进一个数组里，**根本不需要用到 left 和 right 指针！**

```text
       用肉眼看的“大顶堆”              装进物理内存里的“数组”
            100                      索引  0   1   2   3   4
           /   \                     +-----------------------+
         90     80               ->  | 100 | 90| 80| 70| 60| |
        /  \                         +-----------------------+
      70    60
```

如果没有指针，我怎么知道一个节点的左孩子、右孩子和爸爸是谁呢？
数学家给出了三个极其优美的**索引计算公式**（假设当前节点在数组里的下标是 `i`）：
- 我的**左孩子**在：`2 * i + 1`
- 我的**右孩子**在：`2 * i + 2`
- 我的**老爸**在：`(i - 1) / 2`  （向下取整）

只需简单的小学加减乘除，就能在数组里瞬间完成“指针跳转”，这让堆的运行速度极快，且极致节省内存！

## 堆的核心操作：上浮与下沉

既然堆是用数组存的，那我们怎么插入新数据（新病人来排队），又怎么提取最大值（叫最急的病人进去看病）呢？这就涉及两个好玩的动画动作。

### 1. 插入新数据 —— “上浮 (Sift Up)”
假设我们要往上面的大顶堆插入一个超大数字 `110`：
1. **放队尾**：直接把 `110` 放到数组的最后面（变成树的一个新叶子节点）。
2. **往上爬（上浮）**：`110` 一看老爸是 `90`，心想“我的优先级比你高啊”，于是和 `90` **交换位置**；到了新位置再看老爸是 `100`，继续**交换位置**，直到自己成为顶级大老板（根节点）。

### 2. 取出最大值 —— “下沉 (Sift Down)”
由于这是大顶堆，最大的数字永远在根节点（数组下标为 0 的位置）。
1. **抽走根节点**：大老板走了，留下了空位。
2. **拿垫底的顶上**：把坐在数组最后面的、最底层的无名小卒拉上来，强行放到根节点补空位。
3. **往下掉（下沉）**：这个小卒发现底下两个经理（左右孩子）都比自己强，那为了服众，他只能和**最大的那个孩子**交换位置往下掉，一路掉到符合自己实力的层级为止。

这两个操作巧妙地顺着树的高度上下移动，因此时间复杂度都是极其优秀的 **O(log n)**。

## 各语言中的“优先队列”（现成工具）

在实际做软件开发或刷算法题时，我们很少从零手写一个堆（除非面试官强制要求）。绝大多数主流编程语言，都已经为我们内置了写好的优先队列（而且底层大都是用小顶堆或大顶堆实现的）。

请看下面如何用一行代码召唤“优先队列”神器：

{% tabs %}
{% tab title="Python" %}
```python
import heapq

# Python 内置的是小顶堆 (Min-Heap)
# 没有专门的 PriorityQueue 类，而是直接对一个普通 List 进行魔法操作

nums = [5, 1, 8, 3]
heapq.heapify(nums)      # 瞬间把普通数组结构化为小顶堆
print(nums)              # 输出 [1, 3, 8, 5] (不一定是排序的，但 1 一定在最前面)

heapq.heappush(nums, 2)  # 插入一个 2（内部会自动上浮）

smallest = heapq.heappop(nums) # 取出并删除最小值 1（内部会自动下沉）
print(smallest)          # 输出 1
```
{% endtab %}
{% tab title="Java" %}
```java
import java.util.PriorityQueue;

public class Main {
    public static void main(String[] args) {
        // Java 默认内置的是小顶堆 (Min-Heap)
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();

        minHeap.offer(5); // 插入数字
        minHeap.offer(1);
        minHeap.offer(8);
        minHeap.offer(3);

        System.out.println(minHeap.poll()); // 取出并删除最小值，输出 1
        System.out.println(minHeap.peek()); // 只看一眼最小值是谁，不删除，输出 3

        // 如果你想要一个大顶堆？加个反向比较器即可！
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
    }
}
```
{% endtab %}
{% tab title="C++" %}
```cpp
#include <iostream>
#include <queue>
#include <vector>

int main() {
    // C++ 默认内置的是 大顶堆 (Max-Heap)！注意这跟 Java 刚好相反
    std::priority_queue<int> maxHeap;

    maxHeap.push(5);
    maxHeap.push(1);
    maxHeap.push(8);
    maxHeap.push(3);

    std::cout << maxHeap.top() << std::endl; // 输出 8 (最大值)
    maxHeap.pop(); // 删除此时的最大值 (8)

    // 如果要用小顶堆，需要一长串咒语：
    std::priority_queue<int, std::vector<int>, std::greater<int>> minHeap;
    return 0;
}
```

{% tab title="C#" %}
```csharp
using System;
using System.Collections.Generic;

class Program {
    static void Main() {
        // C# 较新版本 (.NET 6+) 引入了 PriorityQueue
        // 第一个是元素类型，第二个是优先级类型。默认是 小顶堆
        PriorityQueue<string, int> pq = new PriorityQueue<string, int>();

        // enqueue 的第一个参数是装的值，第二个参数是决定它排队顺序的"优先级数字"
        pq.Enqueue("张三", 5);
        pq.Enqueue("李四", 1); // 1 最小，优先级最高
        pq.Enqueue("王五", 8);

        // Dequeue 取出优先级数字最小的人
        string first = pq.Dequeue();
        Console.WriteLine(first); // 输出 "李四"
    }
}
```

## 本章小结

- **优先队列**：不是“先来后到”，而是“优先级最高者先出队”。
- **堆 (Heap)**：是实现优先队列的最优底层结构。它逻辑上是一棵**完全二叉树**，但物理上完美**塞在一个无指针的一维数组里**。
- **大顶堆 / 小顶堆**：堆仅仅保证“老总比下属大（或小）”，不保证兄弟之间谁大谁小。
- **动态极值神器**：插入和删除的时间复杂度都是 $O(\log n)$，获取最大值/最小值只需看一眼数组第 0 位，时间复杂度极致的 $O(1)$！
- 面试常考的**“求前 K 大/小元素”**、以及图论中的**“Dijkstra 最短路径算法”**，背后全部仰仗了堆的力量。
