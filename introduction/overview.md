# 数据结构概述

## 什么是数据结构

在编写程序时，我们需要处理各种各样的数据：用户的姓名、商品的价格、地图上的路线。这些数据并不是杂乱无章地堆在一起的，而是需要以某种方式**组织**起来，才能被高效地存取和操作。

数据结构（Data Structure）就是计算机中组织和存储数据的方式。更准确地说，它研究的是数据元素之间的关系，以及在这种关系下如何高效地执行各种操作。

举一个生活中的例子：图书馆里的书可以随意堆在地上，但这样找一本书就会非常困难。如果按照分类号排列在书架上，查找效率就会大幅提升。数据结构所做的事情，本质上就是为数据设计合理的"书架"。

## 为什么要学习数据结构

初学编程时，我们往往只用最简单的变量和数组来解决问题。随着问题规模的增大和需求的复杂化，选择合适的数据结构就变得至关重要。原因有以下几点：

- **效率差异巨大**。同一个问题，使用不同的数据结构，程序的运行速度可能相差成百上千倍。例如，在一百万条记录中查找某个值，用无序数组可能需要逐一比较，而用哈希表几乎可以瞬间完成。
- **代码更易理解和维护**。合适的数据结构能让代码的意图更加清晰。用栈来处理括号匹配，用队列来管理任务调度，都是自然而然的选择。
- **它是算法的基础**。绝大多数算法都依赖于特定的数据结构。不理解树结构，就很难理解二分搜索的变体；不理解图，就无法掌握最短路径算法。

## 数据的逻辑结构

数据结构可以从两个维度来分类：逻辑结构和物理结构。逻辑结构描述的是数据元素之间的抽象关系，与具体的存储方式无关。

### 集合结构

数据元素之间没有特定的关系，仅仅"属于同一个集合"。例如，一个班级的所有学生构成一个集合，学生之间没有先后、层次之分。

### 线性结构

数据元素之间存在**一对一**的关系，形成一条"线"。每个元素（除首尾外）都有且仅有一个前驱和一个后继。

典型代表：数组、链表、栈、队列。

```
元素1 → 元素2 → 元素3 → 元素4
```

### 树形结构

数据元素之间存在**一对多**的关系。一个元素可以有多个下级元素，但每个下级元素只有一个上级。

典型代表：二叉树、堆、B 树。

```
        根
       / \
      A    B
     / \    \
    C   D    E
```

### 图形结构

数据元素之间存在**多对多**的关系。任意两个元素之间都可能存在关联。

典型代表：社交网络、交通网络、依赖关系图。

```
    A --- B
    |   / |
    |  /  |
    C --- D
```

## 数据的物理结构

物理结构（又称存储结构）描述的是数据在计算机内存中的实际存放方式。逻辑结构相同的数据，可能采用不同的物理结构来实现。

### 顺序存储

将数据元素存放在一段**连续的内存空间**中。元素之间的逻辑关系通过物理位置的相邻来体现。

优点是可以通过下标直接访问任意位置的元素（随机访问），缺点是插入和删除操作可能需要移动大量元素。

数组是顺序存储的典型实现。

### 链式存储

数据元素可以存放在**任意的内存位置**，元素之间的关系通过指针（Pointer）来维护。每个元素除了存储自身的数据外，还额外存储指向下一个（或上一个）元素的地址。

优点是插入和删除操作只需修改指针，不需要移动数据；缺点是无法直接通过下标访问，必须从头开始逐个查找。

链表是链式存储的典型实现。

### 两种存储方式的对比

| 比较维度 | 顺序存储 | 链式存储 |
| --- | --- | --- |
| 内存分布 | 连续 | 分散 |
| 随机访问 | 支持，O(1) | 不支持，O(n) |
| 插入/删除 | 需要移动元素，O(n) | 修改指针即可，O(1) |
| 空间利用 | 可能浪费（预分配） | 按需分配，但有指针开销 |

## 数据结构的基本操作

无论使用哪种数据结构，我们对数据的操作通常可以归纳为以下几类：

1. **创建**（Create）— 初始化一个空的数据结构
2. **插入**（Insert）— 向数据结构中添加新元素
3. **删除**（Delete）— 从数据结构中移除指定元素
4. **查找**（Search）— 在数据结构中寻找满足条件的元素
5. **更新**（Update）— 修改数据结构中已有元素的值
6. **遍历**（Traverse）— 依次访问数据结构中的每个元素

不同的数据结构在执行这些操作时的效率各不相同。选择数据结构的核心依据，就是你的程序最频繁地执行哪些操作。如果频繁查找，优先考虑哈希表或二叉搜索树；如果频繁在两端插入删除，优先考虑双端队列或链表。

## 如何选择数据结构

面对一个具体问题时，可以从以下几个方面考虑：

1. **数据量有多大？** 小规模数据下，简单的数组就够用；大规模数据下，需要更高效的结构。
2. **最常执行的操作是什么？** 读多写少选数组，写多读少选链表，查找为主选哈希表。
3. **是否需要保持顺序？** 需要排序选二叉搜索树或有序数组，不需要排序选哈希表。
4. **数据之间有什么关系？** 层次关系用树，网状关系用图，线性关系用数组或链表。

在后续章节中，我们会逐一学习各种数据结构的原理和实现，届时这些选择依据会变得更加具体和直观。

## 本章小结

- 数据结构是组织和存储数据的方式，目的是提高数据操作的效率
- 逻辑结构分为集合、线性、树形、图形四种
- 物理结构分为顺序存储和链式存储两种
- 选择数据结构时，应根据数据规模、操作频率和数据关系综合判断
- 数据结构与算法密不可分，学好数据结构是掌握算法的前提
